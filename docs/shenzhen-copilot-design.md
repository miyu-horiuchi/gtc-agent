## Problem Statement

Hardware founders from the US/Europe going to Shenzhen to prototype and manufacture products face a fragmented, overwhelming ecosystem. Knowledge is scattered across outdated guides, WeChat groups, Reddit threads, and personal networks. There's no single tool that helps them navigate the entire journey from pre-trip logistics to ongoing supply chain management — and actually does things for them rather than just giving advice.

## Demand Evidence

- Founder (Miyu) spent 2 months in Shenzhen during Research at Scale 2026 residency at Chaihuo makerspace
- Multiple Twitter DMs from hardware founders asking for Shenzhen advice post-trip
- Fellow residents at Research at Scale consistently asked for factory contacts, logistics help, and navigation tips
- The Research at Scale survival doc itself exists because this knowledge gap is real — it's a collectively maintained document because no single resource covers it

## Status Quo

Current workarounds are fragmented:
- Bunnie Huang's guide (2016, outdated)
- Cold-messaging factories on Alibaba (hit or miss, scam risk)
- Paying sourcing agents ($5-10k+)
- Asking in hardware Discord/Reddit communities (scattered, unverified advice)
- Research at Scale survival doc (great but limited to residency participants)
- Word of mouth via WeChat groups (hard to access from outside)

## Target User & Narrowest Wedge

**Target user:** Hardware founders from US/Europe planning to visit Shenzhen for prototyping/manufacturing. Both first-timers and experienced, agent adapts depth based on experience level.

**Narrowest wedge (v1):** WhatsApp bot that serves as a smart Rolodex + trip prep assistant:
1. Pre-trip checklist (VPN, eSIM, WeChat, Alipay, transit)
2. Vetted factory contacts for user's specific category
3. Draft intro messages in English + Chinese

## Product Overview

**Shenzhen Co-Pilot** — a WhatsApp chatbot that helps hardware founders navigate the entire prototype-to-production journey in Shenzhen/China.

### Three Core Functions

1. **Action engine** — does things for you:
   - Drafts supplier outreach messages (English + Chinese)
   - Translates WeChat communications
   - Tracks production timelines with reminders
   - Parses BOMs and suggests sourcing
   - Generates pre-trip checklists
   - Provides logistics/navigation info

2. **Knowledge base** — community-sourced, continuously growing:
   - Seed: Miyu's 2-month experience + Research at Scale survival docs
   - Community: Users contribute tips through chat + passive conversation capture
   - Scraped: Alibaba, LCSC, 1688, Made-in-China for supplier data

3. **Network connector** — plugs users into:
   - Vetted factories and contacts
   - WeChat groups (Haptics, Tiles, Circuit Prototyping, Robot Theater)
   - Key people in the ecosystem

### Interface

WhatsApp as primary interface via WhatsApp Business API.

### Trust System (Three Layers)

1. **Reputation-based:** Contributors earn trust over time
2. **Human-reviewed:** Miyu/team verify high-impact tips (factory recs, contacts)
3. **Cross-referenced:** 3+ independent mentions auto-verify

## Technical Architecture

### Tech Stack

- **Interface:** WhatsApp Business API (Twilio or Meta Cloud API)
- **LLM:** Claude API for reasoning, drafting, translation
- **Vector DB:** Pinecone (or Chroma for local dev) for RAG over knowledge base
- **Database:** Postgres for structured data (contacts, BOMs, user state, timelines)
- **Backend:** Python/FastAPI to orchestrate everything
- **Deployment:** Railway or Fly.io

### System Components

```
WhatsApp User
    ↓
WhatsApp Business API (Twilio)
    ↓
FastAPI Backend
    ├── Message Router (intent classification)
    ├── Action Modules
    │   ├── Checklist Generator
    │   ├── Message Drafter (EN/CN)
    │   ├── Translator
    │   ├── BOM Parser
    │   ├── Timeline Tracker
    │   └── Navigation/Logistics
    ├── RAG Engine
    │   ├── Vector DB (knowledge base)
    │   └── Structured DB (contacts, suppliers)
    ├── User State Manager

    │   └── Conversation history
    └── Knowledge Ingestion
        ├── Chat contribution pipeline
        ├── Passive learning pipeline
        └── Scraper pipeline
```

### Data Models

## Build Phases

### Phase 1: MVP (Week 1-2)
- WhatsApp webhook + FastAPI backend
- Claude API integration for chat
- Seed knowledge base ingested (survival doc + your notes)
- RAG for answering questions
- Pre-trip checklist generation
- Basic contact/factory lookup by category

### Phase 2: Action Engine (Week 3-4)
- Message drafting (English + Chinese)
- WeChat message translation
- BOM parsing and sourcing suggestions
- User state tracking (journey phase)

### Phase 3: Community + Trust (Week 5-6)
- Chat-based contribution ("I found a great factory...")
- Passive learning from conversations
- Reputation system for contributors
- Human review queue for high-impact tips
- Cross-reference verification (3+ mentions)

### Phase 4: Advanced (Week 7+)
- Timeline tracking with reminders
- Scraped supplier data (Alibaba, LCSC)
- Production status tracking
- Network features (intro to vetted contacts)

## Success Criteria

1. 10 hardware founders use the bot before/during a Shenzhen trip
2. At least 3 users contribute back knowledge through chat
3. Users report the bot saved them time vs. doing it alone
4. Knowledge base grows beyond seed data
